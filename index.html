Cure your JavaScript fatigue ..become an es-guy

JavaScript landscape has been completely changed — long gone are the days when we used to simply include jQuery in our project and hammered down all the nails with it. Agreeably, with the tons of possibilities around, the JavaScript ecosystem is confusing today. It could easily startle anyone entering or trying to up their game.

I recently scribbled down a set of roadmaps stating how to become a developer in 2017 which clearly hit a nerve: I saw it receive 5k+ stars on github in a few hours (25k now), it was on the frontpage of hacker news for a whole day and was trending on several subreddits. Although it had the roadmaps for backend and devops as well, most of the feedback and questions that I got was regarding JavaScript; people still seemed to be really confused about where and how to start.

With *367th email that I got today asking the same question, I decided to write the concrete, step-by-step outline and start this publication where I will be teaching and helping you conquer the JavaScript ecosystem in a fun and approachable way. We will not only be learning but also we will be building real world applications along the way.

You can either follow the outline yourself or subscribe to this publication recommended

Two weeks of ES6
